项目：HUT二手信息发布平台

这个不涉及钱（狗头）

就算是一个大致的框架吧，毕竟细节也不可能描述得很清楚，然后这个是我个人的一个想法，你们有啥子想法，都给老子提出来啊，咋最近都不吭声啦？（#O。O#）！

用户群

注册

用户在使用本平台之前，需要注册一个账号，需要填写的 text：

  1. 用户名
  2. 手机号
  3. 密码
  4. 确认密码
  5. 验证码

然后的话，如果有可能，我觉得可以整一波学号验证，就是输入学号和姓名啥的，然后拿学校的表格进行匹配，然后经过系统的自动审核

然后那个验证码的话，我有一个想法，就是如果用户注册，会发送验证码到 他 / 她 的手机上，如果这个不能实现的话也没关系，我们就搞一个普通的输入字符的验证码意思下就好了

对了，还有一个很关键的，需要用户同意协议，这个的话其实实现也不是很难，就加一个按钮，加一个弹出文本就好了应该

注：

  • 用户名不能重复
  • 密码长短限制（如果可能的话，可以设置弱密码检测）
  • 手机号正确

再注：

  • 也可以邮箱验证，你们考虑一下

登录

用户在拥有账号的情况下，可进行登录，需要填写的 text：

  1. 用户名（手机号 / 用户名）
  2. 密码
  3. 字符验证码

界面

用户在登陆了之后，可以查看自己的窝

界面要求至少包括：

  1. 个人信息展示
  2. 个人的二手发布信息
  3. 个人的二手求购信息
  4. 我们的团队
  5. 退出登录

可以考虑添加的：

  • 注销用户（这个加不加无所谓）
  • 头像（QAQ）

注：

如果用户点入了我们的团队，就呈上我的邮箱，然后你们也丢一两个邮箱上去，然后用户可以在这里提出意见或者建议

二手信息

发布

发布二手信息时，需要填写的 text：（*）必填，（—）期望填写

  1. 名称（*）
  2. 类型
  3. 图片（*）
  4. 描述（*）
  5. 个人估价（—）
  6. 联系方式（*）

然后再发布之前可以弹出一个告示，就是说要诚信良心啥的（狗头）

广场

成列所有的二手信息，默认按时间顺序从最新发布开始排列

搜索

用户拥有自主搜索选择的权力

分类

二手信息发布的时候，如果选择了类型就归到那一类去，如果没有的话，就归入“杂项”

信息详展

用户再点击了对应二手信息了解详情后，可以得到较为“友善”的二手信息展示页面

系统管理

用户组管理

8说了

超级管理员设置

就是管理员（狗头）

信息处理

就是管后台，以及所有的数据啥的

杂

  • 二手信息应该可以加上评论等功能
  • 要想一想，一个二手信息买卖双方成功谈好之后，我们要怎么得知，然后下架这个信息
  • 然后还有，就是页面少点漏洞，少点BUG。。。
  • 我这个不是很完善，有的地方懒得提了，你们多多补充
  • 我初步想法是四月初开始这个项目（如果是在五月份前交的话，可能会有略微提早）
